# DOCUMENT ĐẶC TẢ WEBSITE

## Tên dự án

**Giải mã bức ảnh lịch sử**

## Chủ đề nội dung

**Việc cải cách ruộng đất năm 1953–1956 Đảng ta đã mắc sai lầm như thế nào?**

## Mục tiêu sản phẩm

Xây dựng một website game phục vụ **thuyết trình trên lớp**, trong đó cả lớp cùng tham gia theo nhóm.
Nhóm thuyết trình là **host/MC**, điều khiển game trên laptop và chiếu lên TV hoặc màn hình lớn.

Website không phải là game cho từng người chơi tự dùng điện thoại.
Đây là **game điều phối tập thể**, nơi:

* cả lớp nhìn chung một màn hình,
* các nhóm giơ tay giành quyền trả lời,
* host tự xác nhận đúng/sai,
* hệ thống tự mở mảnh ghép và cộng điểm.

# 1. Tổng quan game là gì

“Giải mã bức ảnh lịch sử” là một game đoán hình ảnh theo nhóm.

Game gồm **2 sự kiện lịch sử**.
Mỗi sự kiện có:

* 1 ảnh tư liệu lịch sử,
* ảnh được chia thành **9 mảnh ghép** theo lưới **3x3**,
* mỗi mảnh ghép tương ứng với **1 câu hỏi**.

Luật chơi:

1. Các nhóm lần lượt chọn một ô.
2. Host đọc câu hỏi của ô đó.
3. Nhóm nào được quyền trả lời sẽ trả lời trực tiếp bằng miệng.
4. Host bấm:

   * **Đúng** → ô được mở và nhóm được cộng điểm.
   * **Sai** → ô không mở, không cộng điểm.
5. Khi đã mở một số mảnh, nhóm có thể đoán **đó là sự kiện gì**.
6. Nếu đoán đúng, nhóm đó được **điểm thưởng cao hơn điểm từng mảnh ghép**.
7. Kết thúc sự kiện 1 thì chuyển sang sự kiện 2.
8. Hết 2 sự kiện thì hiển thị bảng xếp hạng chung cuộc.

Game này phải tạo được cảm giác:

* dễ chơi,
* nhanh,
* rõ luật,
* dễ chiếu lên TV,
* lớp học tương tác tốt,
* nội dung lịch sử được dẫn dắt tự nhiên.

# 2. Mục tiêu học thuật của game

Game không chỉ để vui mà phải phục vụ thuyết trình.

Sau khi chơi xong, người học phải nắm được:

* cải cách ruộng đất là một chủ trương lớn trong bối cảnh lịch sử lúc đó,
* sai lầm chủ yếu nằm ở **lãnh đạo, chỉ đạo và tổ chức thực hiện**,
* các sai lầm nổi bật gồm:

  * áp dụng máy móc,
  * phân loại giai cấp sai,
  * mở rộng diện quy kết,
  * vi phạm pháp chế và dân chủ,
  * dùng biện pháp đấu tranh quá mức,
* Đảng đã **thẳng thắn kiểm điểm, nhận sai và sửa sai**.

Website không cần giải thích toàn bộ kiến thức ngay từ đầu.
Nội dung phải được truyền tải dần qua:

* câu hỏi,
* mảnh ảnh được mở,
* phần công bố đáp án,
* lời chốt của host.

# 4. Kiểu triển khai được yêu cầu

## Kiến trúc đề xuất

Dùng **NestJS monolith**.

NestJS sẽ:

* phục vụ API
* quản lý state của game
* phục vụ frontend tĩnh hoặc template-rendered pages

## Khuyến nghị triển khai

Có 2 route chính:

* `/host`
  Dành cho nhóm thuyết trình điều khiển game

* `/display`
  Dành cho màn hình TV / máy chiếu

Nếu chỉ có 1 laptop cắm HDMI lên TV thì có thể dùng:

* chỉ mở `/host`

Nếu muốn đẹp và chuyên nghiệp hơn:

* laptop host mở `/host`
* TV mở `/display`

Hai màn hình phải đồng bộ cùng một session game.

# 6. Luồng chơi chuẩn

Ví dụ:

Sau đó bấm “Tạo phiên chơi”.

Hiển thị:

* tên sự kiện 1 dưới dạng ẩn
* ảnh bị che bởi 9 ô
* danh sách các nhóm và điểm hiện tại
* nút chọn ô

## 6.4 Chọn ô

Khi host chọn một ô:

* website mở popup câu hỏi
* popup hiển thị:

  * mã ô
  * nội dung câu hỏi
  * nếu là trắc nghiệm thì hiển thị A/B/C/D
  * nút “Đúng”
  * nút “Sai”
  * chọn nhóm được cộng điểm

## 6.5 Trả lời đúng

Khi host bấm “Đúng”:

* ô đó được mở
* ảnh phía sau hiện ra đúng phần của ô
* nhóm được cộng điểm
* có animation mở ô
* có thông báo “Nhóm X trả lời đúng”

## 6.6 Trả lời sai

Khi host bấm “Sai”:

* ô không mở
* không cộng điểm
* có thể hiển thị đáp án đúng ngắn gọn
* game quay về trạng thái chờ chọn ô tiếp theo

## 6.7 Đoán sự kiện

Bất kỳ lúc nào sau khi ít nhất 1 ô đã mở, host có thể bấm “Đoán sự kiện”.

Popup đoán sự kiện gồm:

* chọn nhóm đoán
* nhập tên sự kiện hoặc chọn từ danh sách đáp án cấu hình sẵn
* nút xác nhận đúng / sai

Nếu đúng:

* cộng điểm thưởng tùy số ô đã mở
* mở toàn bộ ảnh
* hiển thị tên sự kiện
* hiển thị mô tả ngắn về ý nghĩa lịch sử
* cho phép sang vòng tiếp theo

Nếu sai:

* không cộng điểm
* quay lại màn chính
* tiếp tục chọn ô

## 6.8 Kết thúc vòng

Kết thúc khi:

* có nhóm đoán đúng sự kiện, hoặc
* cả 9 ô đã mở

Sau đó hiển thị:

* tên sự kiện
* ảnh đầy đủ
* mô tả tóm tắt
* nút “Sang sự kiện tiếp theo”

## 6.9 Kết thúc game

Sau khi xong 2 sự kiện:

* hiển thị bảng điểm cuối

# 14. Game state cần quản lý

Hệ thống cần quản lý trạng thái phiên chơi.

## Session state

```ts
type GameStatus =
  | 'setup'
  | 'ready'
  | 'playing'
  | 'question_open'
  | 'guess_open'
  | 'event_completed'
  | 'game_completed';
```

## State tổng

```ts
interface GameSessionState {
  id: string;
  title: string;
  status: GameStatus;
  currentEventIndex: number;
  teams: TeamState[];
  events: EventState[];
  selectedTileId: string | null;
  lastActionMessage: string | null;
  createdAt: string;
  updatedAt: string;
}
```

## Team state

```ts
interface TeamState {
  id: string;
  name: string;
  score: number;
}
```

## Event state

```ts
interface EventState {
  id: string;
  title: string;
  shortDescription: string;
  imageUrl: string;
  answerKeywords: string[];
  isCompleted: boolean;
  tiles: TileState[];
}
```

## Tile state

```ts
interface TileState {
  id: string;
  index: number; // 1..9
  questionType: 'multiple_choice' | 'true_false' | 'short_answer';
  question: string;
  options?: string[];
  correctAnswer: string | boolean | string[];
  explanation?: string;
  revealed: boolean;
  attempted: boolean;
}
```

# 16. API đề xuất cho NestJS

## 16.1 Session APIs

### Tạo session

`POST /api/game/session`

Request:

```json
{
  "title": "Giải mã bức ảnh lịch sử",
  "teamNames": ["Nhóm 1", "Nhóm 2", "Nhóm 3", "Nhóm 4"]
}
```

Response:

```json
{
  "sessionId": "abc123",
  "status": "ready"
}
```

### Lấy state hiện tại

`GET /api/game/session/:sessionId`

### Reset session

`POST /api/game/session/:sessionId/reset`

---

## 16.2 Event APIs

### Bắt đầu event hiện tại

`POST /api/game/session/:sessionId/start`

### Sang event tiếp theo

`POST /api/game/session/:sessionId/next-event`

### Kết thúc game

`POST /api/game/session/:sessionId/finish`

---

## 16.3 Tile APIs

### Chọn tile

`POST /api/game/session/:sessionId/select-tile`

Request:

```json
{
  "eventId": "event-1",
  "tileId": "tile-3"
}
```

### Chấm câu hỏi

`POST /api/game/session/:sessionId/judge-answer`

Request:

```json
{
  "eventId": "event-1",
  "tileId": "tile-3",
  "teamId": "team-2",
  "isCorrect": true
}
```

Hành vi:

* nếu `isCorrect = true` → reveal tile + cộng điểm
* nếu `false` → chỉ mark attempted

## 16.4 Guess APIs

### Mở mode đoán sự kiện

`POST /api/game/session/:sessionId/open-guess`

### Chấm đoán sự kiện

`POST /api/game/session/:sessionId/judge-guess`

Request:

```json
{
  "eventId": "event-1",
  "teamId": "team-1",
  "guessText": "Luật Cải cách ruộng đất năm 1953",
  "isCorrect": true
}
```

Nếu `isCorrect = true`:

* cộng điểm thưởng theo số tile đã reveal
* mark event completed
* reveal full image

# 17. WebSocket Gateway đề xuất

Nếu dùng 2 màn hình `/host` và `/display`, nên dùng WebSocket.

## Gateway

`GameGateway`

Phát các event:

* `session.updated`
* `tile.selected`
* `question.opened`
* `tile.revealed`
* `answer.judged`
* `guess.opened`
* `guess.judged`
* `event.completed`
* `score.updated`
* `game.completed`

Mỗi khi host thao tác ở `/host`, `/display` phải update gần như ngay lập tức.

Nếu Claude muốn đơn giản hơn, có thể dùng polling 1–2 giây/lần.
Nhưng khuyến nghị dùng **Socket.IO với NestJS Gateway**.

# 26. Acceptance Criteria

## AC-01

Host có thể tạo phiên chơi với ít nhất 2 nhóm.

## AC-02

Game hiển thị đúng 2 sự kiện.

## AC-03

Mỗi sự kiện có 9 ô.

## AC-04

Mỗi ô có thể được chọn và mở popup câu hỏi.

## AC-05

Host có thể chấm đúng/sai thủ công.

## AC-06

Nếu đúng, ô được mở và cộng 10 điểm.

## AC-07

Nếu sai, ô không mở và không cộng điểm.

## AC-08

Host có thể mở popup đoán sự kiện bất cứ lúc nào sau khi đã mở ít nhất 1 ô.

## AC-09

Nếu đoán đúng, hệ thống cộng điểm thưởng theo số ô đã mở.

## AC-10

Sau khi đoán đúng, ảnh được mở toàn bộ và sự kiện được đánh dấu hoàn thành.

## AC-11

Sau sự kiện 1, host có thể chuyển sang sự kiện 2.

## AC-12

Kết thúc 2 sự kiện, hệ thống hiển thị bảng điểm cuối.

## AC-13

Nếu dùng `/host` và `/display`, hai màn hình phải đồng bộ.

## AC-14

Host có thể reset toàn bộ game.
